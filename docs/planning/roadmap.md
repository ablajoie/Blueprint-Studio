# Product Principles

These should evolve over time.

- Clarity over completeness.
- Discovery is a workspace, not a document.
- Every screen answers one primary question.
- Reduce context switching.
- Minimize scrolling whenever practical.
- Keyboard-first where appropriate.
- Progressive disclosure over overwhelming interfaces.
- Capture decisions where they naturally occur.
- Build the 80% users need every day, and make it exceptional.
- Blueprint complements delivery tools rather than replacing them.
- Keep developers focused on development by improving solution readiness.
