Product Principles

This is how you decide whether a feature belongs.

Not requirements.
Principles.

These are our filters.
Every feature gets tested against them.

Clarity over completeness.
If a feature makes the product harder to understand, reconsider it.

Reduce context switching.
Users should stay inside Blueprint whenever practical.

Discovery is a workspace.
Not a document.

Capture decisions naturally.
Don't force users to create decision records.
Capture them as work happens.

Build the 80%.
Don't recreate Excel.
Don't recreate Figma.
Deliver the 80% users actually need.

Every page answers one question.
Discovery answers
"What problem are we solving?"

Metadata answers
"What are we building?"

Requirements answer
"What must it do?"
